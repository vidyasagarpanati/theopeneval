# The OpenEval Principles
## foundational and rarely change
### Version 0.1
### Dated: 5th June, 2026

⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻

# Principle 1: Reliability Before Capability

AI capability is meaningless without reliability.
An AI system that is occasionally brilliant but frequently wrong cannot be trusted.
OpenEval prioritizes reliability over demonstrations.

⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻

# Principle 2: Evaluation Is Continuous

Evaluation is not an event. It is a continuous process.
Every change to an AI system introduces the possibility of regression.
Reliability must be continuously measured.

⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻

# Principle 3: Everything Is Measurable

If reliability cannot be measured, it cannot be improved.
OpenEval exists to make AI behavior observable and measurable.

⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻

# Principle 4: Trust Requires Evidence

Trust should not be based on:
* Marketing
* Anecdotes
* Demos

Trust should be based on evidence. Evaluations provide that evidence.

⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻

# Principle 5: Reliability Is Multi-Dimensional

Accuracy alone is insufficient. 
Reliability includes:
* Accuracy
* Robustness
* Consistency
* Cost
* Latency
* Safety
* Tool Usage
* Task Completion

⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻

# Principle 6: Open Standards Accelerate Progress

Vendor-specific evaluation creates fragmentation.
Open standards create interoperability.

⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻

# Principle 7: Evaluation Must Be Reproducible

Results must be reproducible across:
* Teams
* Models
* Frameworks
* Environments

⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻

# Principle 8: Evaluation Must Be Transparent

Scores without explanations create false confidence.
Every reliability signal should be traceable.

⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻

# Principle 9: Reliability Is A System Property

Models do not fail. Systems fail.
Evaluation must assess:
Model + Prompt + Tools + Memory + Workflow + Environment
not merely the model.

⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻⸻

# Principle 10: Community Over Vendor Control

AI Reliability Engineering should belong to the ecosystem.
Not a single company.
